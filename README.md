# 运营商投诉分类治理项目

## 项目要做什么

这个项目面向运营商历史投诉工单，目标是建设一套可持续优化的投诉治理基础能力，包含：

- 稳定的投诉分类体系
- 多维度的标签体系
- 支撑 AI 批量分类和打标的数据库结构
- 从历史工单中沉淀规则、关键词和推荐关系的机制
- 面向新工单的自动处理建议与后续编排能力

当前数据源核心是历史工单表 `raw_complaint_tickets`，主键为 `ticket_id`。

## 总体流程

### 第一阶段：数据库与主数据建设

先完成数据库基础建设，明确：

- 分类表
- 标签组与标签表
- 分类标签基线关系表
- 分类规则表与标签规则表
- AI 结果沉淀表
- 评估与统计相关表

这一步的目标是把“分类是什么、标签是什么、结果怎么存、规则怎么长出来”先定义清楚。

### 第二阶段：历史工单批量 AI 标注

让 AI 读取历史工单的完整信息，输出：

- 分类结果
- 标签结果
- 关键词结果
- 命中依据

这一步的目标是验证当前分类和标签体系是否合理，并形成第一批可分析结果。

### 第三阶段：评估与反馈优化

基于历史工单标注结果，持续观察：

- 分类命中率
- 标签命中率
- 高频关键词
- 高频分类-标签共现关系
- 规则有效性

再把这些结果反向沉淀到规则表和基线关系表里，优化分类与标签体系。

### 第四阶段：AI 处理新工单

当体系稳定后，新工单进入时采用混合判定流程：

- AI 判断主分类
- AI 抽取多标签
- 结合基线关系和规则做校验或补充
- 根据分类 + 标签生成处理建议、派单建议和风险提示

最终目标不是只做“自动分类”，而是做一套完整的投诉治理闭环。
