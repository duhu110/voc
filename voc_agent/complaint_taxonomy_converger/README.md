# complaint_taxonomy_converger

## 目标

`complaint_taxonomy_converger` 是面向“收敛判断”的新 agent。

它不追求一次输出很多候选，而是优先回答两个问题：

1. 这条工单能不能稳定命中一个主分类
2. 在已确认分类的前提下，每个标签组里能不能稳定命中至多一个标签

当前版本不落数据库，只做验证性输出与批量统计。

## 当前流程

1. `load_context`
   - 读取单条原始工单
   - 读取启用中的分类清单
   - 读取启用中的标签清单
2. `analyze_category`
   - 只允许返回一个主分类
   - 必须满足分类置信度阈值
   - 如果没有找到可信分类，则直接停止
3. `analyze_tags`
   - 只有分类成功后才执行
   - 每个标签组最多保留一个标签
   - 标签也必须满足标签置信度阈值
   - 没有标签不算失败
4. `finalize_result`
   - 统一输出收敛版结果结构

## 当前约束

- 分类最多 1 个
- 标签每组最多 1 个
- 分类找不到就跳过工单
- 标签找不到允许为空
- 当前不做数据库写入
- 当前不做关键词抽取与去重入库

## 目录结构

- `chain.py`: LangGraph 组织入口
- `prompts.py`: 分类与标签的 prompt 模板
- `state.py`: state 与结构化输出模型
- `nodes/`: 图节点
- `tools/`: agent 专用工具
- `utils/`: 输出归一化和收敛裁剪
- `tests/`: 新 agent 的测试
- `scripts/`: 常用调试和批量统计脚本

## 本轮验证目标

先跑 20 条工单，不落数据库，只统计：

- 找到分类的工单数
- 找到至少一个标签的工单数
- 因分类不达标而跳过的工单数

## 常用脚本

- `scripts/run_single_ticket.py`
  - 单条工单验证
- `scripts/run_sample_batch.py`
  - 抽样工单并统计分类/标签命中情况
